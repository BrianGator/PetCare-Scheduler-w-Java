# PetCare Scheduler - Development Summary

## Project Overview
The PetCare Scheduler is a Java-based console application designed for managing pet health records and scheduling appointments. It provides a robust command-line interface for registering pets, scheduling care activities, persisting data to disk, and generating proactive health reports.

## Technologies Used
- **Core Language**: Java (JDK 8+)
- **Date/Time Management**: `java.time` package (LocalDateTime, LocalDate)
- **Data Persistence**: Java Object Serialization (`java.io.Serializable`)
- **User Interface**: Console-based CLI using `java.util.Scanner`
- **Data Structures**: Java Collections Framework (`ArrayList`, `Stream API` for reporting)

## File Summary
- **Appointment.java**: The blueprint for individual care events. Stores appointment type, date/time, and clinical notes.
- **Pet.java**: The primary data container for pet profiles and their associated history of appointments.
- **PetCareScheduler.java**: The main application controller. Handles the UI loop, file I/O operations, business logic validation, and report generation logic.
- **metadata.json**: Application metadata for the development environment.
- **package.json**: Environment configuration for the project container.

## Completion Statement
I have completed the development of the PetCare Scheduler Java console application. I have created the requested object blueprint classes and the main application logic, incorporating data persistence, reporting, and robust error handling.

### Key Components Built:

**Appointment.java:**
- Defines the structure for appointments (type, date/time, notes).
- Provides full getters, setters, and a formatted `toString()` method for clean console output.

**Pet.java:**
- Contains pet profiles (unique ID, name, breed, age, owner info).
- Maintains an internal list of Appointment objects specifically for each pet.

**PetCareScheduler.java:**
- **Main Menu**: A robust console interface using a switch-case loop.
- **Data Persistence**: Implements Serializable to save and load pet data to/from `pets_data.ser`, ensuring records are preserved between sessions.
- **Validation**:
  - Prevents duplicate Pet IDs.
  - Ensures appointment dates are in the future.
  - Validates age and numeric inputs to prevent crashes.
- **Reporting**:
  - Generates a list of appointments scheduled for the next 7 days.
  - Identifies pets that haven't had a "vet visit" in the last 6 months.
- **Record Display**: Allows viewing all pets, specific pet histories, or upcoming schedules across the entire database.

### How to Run:
Since this is a Java console application and we are in a limited development environment, you can compile and run these files locally on any machine with a JDK (Java Development Kit) installed:

**Compile:**
```bash
javac *.java
```

**Run:**
```bash
java PetCareScheduler
```

The application will automatically create a `pets_data.ser` file in the same directory once you store data or exit.
